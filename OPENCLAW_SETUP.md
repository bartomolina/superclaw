# Recommended SuperClaw setup

This file is for the **recommended initial environment** around SuperClaw, not the SuperClaw app install itself.

The idea is to document the setup that tends to work best before or alongside installing SuperClaw.

## Suggested baseline

- run OpenClaw on a dedicated Linux host (for example Hetzner)
- get the base OpenClaw install working first
- set up Git and GitHub access early (`git`, `gh`, repo auth, SSH keys if desired)
- set up Cloudflare Tunnel for app exposure/management
- prepare Convex and Resend before enabling Kanban

This is currently a first-pass placeholder and can be expanded as the preferred setup gets nailed down.
